# Состояние проекта — технический хендофф

_Обновлено: 2026-07-13. Пилотный проект коттеджного посёлка, н.п. Большое
Афанасово, Нижнекамский р-н, РТ._

## 0. Цель

Автоматизированно готовить комплект градостроительных чертежей концепт-стадии (ПП)
по топосъёмке заказчика: DWG → редактируемый DXF (МСК-16) + альбом листов + ТЭП +
3D-визуализация. Ориентир качества — исходный комплект по этому же участку
(папка `результат/`), который отдан только в PDF; наша задача — превзойти его по
полноте и отдать редактируемые файлы. Рабочее CAD-окружение проверялось на
**Компас-3D** и **SketchUp** (исходный комплект) и на **nanoCAD / Blender**
(наш пайплайн, Linux/Arch).

## 1. Что на входе

## 4. Инструменты и окружение

- **LibreDWG собран из исходников в `~/.local`** (`~/.local/bin/dwg2dxf`) —
  **сохраняется между сессиями** (без root, Arch). Читает DWG→DXF.
- **Python venv — ЭФЕМЕРНЫЙ** (был в scratchpad сессии, пропадёт). Пересоздать:
  `python3 -m venv venv && venv/bin/pip install ezdxf shapely numpy matplotlib pillow`.
- `ffmpeg`, `node` — есть в системе. **Blender НЕ установлен** (нет root) — рендеры
  и облёт делались через matplotlib+ffmpeg; фотореализм — только у пользователя в Blender.
- `pipeline/setup.sh` — ставит venv и (при необходимости) собирает LibreDWG.

## 5. Как всё воспроизвести (скрипты сохранены в `pipeline/`)

- `pipeline/generate_genplan.py` — **главный параметрический генератор**: DWG →
  генплан.dxf + СПОЗУ.pdf + ТЭП. Одна команда (см. `pipeline/README.md`).
- `pipeline/generators/` — 30 рабочих скриптов листов/3D (перенесены из
  эфемерного scratchpad): `gen*.py` (генплан), `compose*.py` (листы СПОЗУ/сети),
  `gen_facades.py` `gen_situational.py` `gen_cover.py` `gen_house.py`
  `gen_sections.py` (листы 9/2/1/8/6), `build_terrain.py` `build_village.py`
  `build_house3d.py` `flythrough.py` (3D+видео), `export_scene.py` (сцена прогулки),
  `dxf_export.py` `add_utils_dxf.py`, `_patch.py`.
- `pipeline/3d/` — 3D-скрипты + `blender_setup.py` (авто-настройка Blender,
  **не тестировался** — Blender отсутствует) + `Blender_гайд.md`.
- `pipeline/web/` — `template.html` + `build_site.py` (сборка альбома),
  `walk_tpl.html` (прогулка), `three/` (three.min.js r128 + PointerLockControls),
  `assets/` (оптимизированные jpg + mp4).
- `pipeline/data/` — промежуточные данные, чтобы **не пересчитывать с нуля**:
  `boundary.json` (проектная граница), `plan.pkl` (геометрия генплана),
  `dem.npz` (ЦМР рельефа), `utils.pkl` (сети/дороги), `tep.pkl`, `scene.json`.

Сборка альбома: положить `pipeline/data`, `pipeline/web/assets`, `pipeline/web/three`
рядом → `venv/bin/python build_site.py` → `artifact.html`. Прогулка:
`export_scene.py` → `scene.json`, затем вставить three.min.js+PLC+scene.json в
`walk_tpl.html` (см. как в build-логике).

## 6. Ключевые технические факты и грабли

- **Проектная граница** = замкнутый red-контур в DWG: слой `-_линия`, цвет ACI
  `1`, 111 вершин, 17,69 га. Извлечение: `--boundary-color 1 --target-ha 18`.
- **Чтение DWG через ezdxf**: нужен `ezdxf.recover.readfile` + monkeypatch
  `_patch.py` (глушит `DXFInvalidLineType` на пустых linetype из LibreDWG).
- **three.js в артефактах**: DoubleSide на материалах (иначе грани отсекаются и
  рельеф/дома невидимы). Мапинг мир→three: `x=wx, y=elev, z=-wy`.
- **⚠ Pointer Lock в артефактах (iframe-песочница) ЗАБЛОКИРОВАН** — управление
  взглядом в прогулке сделано через **drag-look (зажатая мышь)**, НЕ через
  `PointerLockControls`. Старт камеры повёрнут к ближайшим домам.
- **Кодировка**: в самодостаточные HTML добавлен `<meta charset="utf-8">`.
- Палитра графиков ТЭП прогнана через валидатор dataviz (CVD/контраст):
  `#1f6fb2,#9c7018,#7a4fb0,#4e8a2f`.
