# Пайплайн: топосъёмка DWG → концепт-генплан

Автоматически строит генплан коттеджного посёлка из топосъёмки: читает DWG,
находит проектную границу, нарезает кварталы и участки, сажает дома всех типов,
считает ТЭП и выдаёт **редактируемый DXF (МСК-16) + лист СПОЗУ (PDF/PNG)**.

## Установка (один раз)

```bash
bash pipeline/setup.sh
```

Ставит Python-окружение (`ezdxf`, `shapely`, `numpy`, `matplotlib`) и, если нужно,
собирает **LibreDWG** (`dwg2dxf`) в `~/.local` — без прав root. Требуются `gcc`,
`autotools`, `swig` (для сборки LibreDWG). Если подаёте сразу `.dxf`, LibreDWG не нужен.

## Запуск на новом участке

```bash
pipeline/venv/bin/python pipeline/generate_genplan.py \
    путь/к/топосъёмке.dwg --out результат_нового_проекта \
    --boundary-color 1 --target-ha 18
```

Полезные параметры:

| Параметр | Зачем |
|---|---|
| `--boundary-layer ИМЯ` | если граница на известном слое |
| `--boundary-color N`   | цвет ACI линии границы (у Афанасово — `1`, красный) |
| `--target-ha N`        | ориентир площади участка, помогает выбрать нужный контур |
| `--front / --depth / --road` | ширина участка по фронту / глубина / полуширина проезда, м |

Если граница не нашлась — откройте топосъёмку и посмотрите слой/цвет контура
проектной территории, передайте их флагами.

## Что на выходе

- `Генплан_МСК16.dxf` — редактируемый чертёж по слоям (граница, дороги, участки,
  дома по типам, соцобъекты). Открывается в nanoCAD / AutoCAD, ложится на съёмку.
- `СПОЗУ.pdf` / `.png` — оформленный лист с ТЭП.
- `ТЭП.json` — показатели в машиночитаемом виде.

## Границы применимости (честно)

Это **концепт-стадия (ПП)**, стартовая заготовка для доводки человеком в CAD, а не
рабочая документация. Регулярная сетка не оптимизирована под рельеф; трассировку
улиц по горизонталям, планы этажей/фасады домов (из каталогов типовых проектов) и
детальную раскладку сетей доделывает проектировщик. Сводный план сетей и разрезы
дорог для проекта Афанасово сделаны отдельными скриптами тем же методом
(см. `результат_v2/`).
