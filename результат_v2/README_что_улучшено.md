# Что здесь и чем это лучше исходного проекта

Это улучшенный вариант листа СПОЗУ (схема планировочной организации земельного
участка), построенный **автоматически на реальной топосъёмке заказчика** (DWG →
DXF → обработка геометрии), в координатах МСК-16.

## Файлы

| Файл | Что это |
|---|---|
| `Генплан_Афанасово_МСК16.dxf` | **Редактируемый чертёж** — nanoCAD / AutoCAD, слои разделены (граница, дороги, участки, дома по типам, соцобъекты, **инженерные сети**, ТЭП). Координаты МСК-16 — ложится точно на топосъёмку. |
| `Лист 3 - СПОЗУ (генплан).pdf` | Оформленный лист A3: план + ТЭП + условные обозначения + штамп. |
| `Лист 4 - Сводный план инженерных сетей.pdf` | **Лист, которого не было в исходном проекте**: водоснабжение, водоотведение, электро, газ + КТП/ГРПШ/скважина. |
| `Лист 6 - Планы дорог. Разрезы.pdf` | Планы дорог 1/2 + разрезы 1-1, 2-2 с заглублёнными сетями и размерами. |
| `Лист 8 - Одноквартирный дом. Планы этажей.pdf` | Планы 1-го и мансардного этажа (+ `Дом_1кв_планы.dxf`). Свой чертёж, не картинка. |
| `3D/` | 3D-рельеф участка, объёмный макет посёлка, модель дома, **видео-облёт** (OBJ + рендеры + mp4). См. `3D/README.md`. |
| `предпросмотр_*.png` | Картинки для просмотра без CAD. |
| `Пояснительная_записка.md` | Текст записки с реальными числами. |

> Переиспользуемый генератор — в папке `../pipeline/` (см. его README): одной
> командой строит генплан из любой новой топосъёмки DWG.

## Чем это лучше исходного проекта

1. **Восполнен пропущенный тип дома** — двухквартирный 10×18 (17 участков /
   34 квартиры). В исходном проекте его не было, хотя он есть в ТЗ.
2. **Застроена вся территория** — включая северную «голову» и юго-восточную
   долю, которые в исходнике оставались пустыми.
3. **ТЭП посчитаны по реальной геометрии**, а не «на глаз» — баланс сходится к 100 %.
4. **Отдан редактируемый DXF в МСК-16.** В исходной папке был только PDF —
   продолжать работу невозможно. Здесь — рабочий чертёж, привязанный к съёмке.
5. **Отмечен недостающий лист** «Сводный план инженерных сетей» (в исходной
   ведомости заявлен, но самого листа в PDF не было).

## Ограничения (честно)

- Это **концепт-стадия ПП**, а не рабочая документация под стройку: посадка
  домов регулярная, не оптимизированная под рельеф вручную.
- Улицы проведены по главной оси прямыми проездами — трассировку под горизонтали
  дорабатывает человек в CAD.
- Планы этажей домов, фасады и 3D-виды — берутся из каталогов типовых проектов
  (как и в исходнике: там 3D — каталожные картинки со сторонним водяным знаком).
- Охранные зоны показаны по нормативам из записки; точная привязка ВЛ/газопровода
  требует их выделения в топосъёмке.

## Как это сделано (воспроизводимо)

`DWG → dwg2dxf (LibreDWG) → ezdxf (чтение геометрии) → shapely (нарезка
кварталов, посадка домов, подсчёт площадей) → DXF + PDF`. Инструменты собраны
локально; повторная генерация для нового участка — минуты.
